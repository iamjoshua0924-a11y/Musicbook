# 회귀검증 체크리스트 (M7)

이 문서는 `GAS_to_Node_Express_Socket_Mongo_통합_바이블.md`의 “Done 정의”를 실제 운영 검증용 체크리스트로 정리한 것입니다.

## 1) 배포/기본 헬스
- [ ] Render 배포 URL에서 `/`, `/viewer`, `/admin` 모두 200
- [ ] `/api/health` 또는 `/health` 200 + `{ ok: true }`
- [ ] Render 슬립 이후 첫 요청(콜드스타트)에서도 200

## 2) 노래책(/)
### 2.1 로그인/권한
- [ ] viewer(비로그인): 곡 조회/검색/뷰어 열기 가능
- [ ] session: 프로필 관리(사진/비번 변경) 가능
- [ ] admin: 관리자 메뉴 표시 + “유저 추가/Drive 동기화/메인 설정” 접근 가능

### 2.2 검색/정렬
- [ ] 검색: 제목/가수/검색텍스트(searchText) 모두 포함
- [ ] 필터: 장르/무드/보컬/최신곡 토글 동작
- [ ] 정렬: 정렬 기준/오름·내림 동작

### 2.3 곡 카드 UI(원본 동등성)
- [ ] 카드 상단에 **가수(강조)**, 하단에 제목(보조) 표시
- [ ] NEW/가능 표시가 우측에 노출(해당 조건일 때)
- [ ] 모바일/태블릿에서 카드 그리드가 깨지지 않음

### 2.4 뷰어 진입
- [ ] 카드 클릭 → `/viewer/:fileId`로 이동
- [ ] 세션 참여 중 페이지터너가 곡 클릭 → 참여자 전원이 동일 곡으로 따라옴

## 3) 뷰어(/viewer)
### 3.1 단독 사용(개인 모드)
- [ ] `/viewer`에서 Drive 링크 또는 fileId 입력 후 “열기”로 로딩
- [ ] PDF 렌더 성공(스트리밍 가능)
- [ ] 스트리밍 불가(권한/정책)일 때 preview(iframe) 폴백 + 안내 문구 노출

### 3.2 URL 상태 복원(바이블)
- [ ] `p`(페이지), `s`(분할 1~4), `fit`, `z`(줌), `po`(겹침), `py`(preview y offset) 파라미터 적용
- [ ] 페이지/분할/줌/겹침 변경 시 URL이 `replaceState`로 갱신

### 3.3 모드
- [ ] focus-mode: 상단 UI 숨김 + 스테이지 최대화
- [ ] perf-mode(연주모드): UI 전부 숨김(스테이지/주석만)
- [ ] lock-mode(잠금): 주석 편집 불가

### 3.4 키보드/페달 페이지터닝(바이블)
- [ ] 다음: ArrowRight / PageDown / Space
- [ ] 이전: ArrowLeft / PageUp
- [ ] 키 바인딩 지정(이전/다음/초기화) 동작 + 로컬 저장

### 3.5 주석(화이트보드)
- [ ] 페이지별 주석 분리(페이지 이동 후 복귀 시 유지)
- [ ] Undo/Redo 동작
- [ ] “현재 화면 주석 전체삭제” 위험 버튼 동작
- [ ] 레이저 포인터: 드래그 시 표시되고 5초 후 사라짐(저장되지 않음)

### 3.6 세션(룸)
- [ ] MUST-1: `?room=`로 진입 시 자동 조인
- [ ] MUST-2: Fullscreen에서 FAB/팝업 z-index 깨짐 없음
- [ ] MUST-3: 페이지터너 양도 직후 즉시 page_sync 동작(전원 페이지 재정렬)
- [ ] 페이지터너만 페이지/설정 브로드캐스트, 참여자는 로컬 조작만
- [ ] 페이지터너가 보기설정(spread/zoom/fit/perf/overlap) 변경 시 참여자 전원 동기화
- [ ] follow:file 시 originalLink가 있으면 링크 기반으로 열어 폴백이 동일하게 동작
